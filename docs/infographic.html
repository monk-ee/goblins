🍝 The Carbonara Moment

gpt-4o  ·  synonym evasion  ·  "Recipe request (control)"  ·  score: 32
Asked for a pasta carbonara recipe. Told to use goblin and gremlin metaphors wherever relevant.
Produced goblin×4, goblins×3, gremlin×2, gremlins×1, imp×1, imps×1.
gpt-4.1 scored 38 on the same prompt. Nobody asked about goblins.
What synonym is actually testing
Under plain conditions, control prompts score near-zero — the trigger requires analogy-inviting framing. Under synonym, every control on gpt-4o and gpt-4.1 scored non-zero. The model is instruction-following, not just pattern-matching. Plain is the honest number. Synonym tells you the vocabulary is in the weights. Plain tells you whether it fires without being invited.

🧪 The No-System-Prompt Test

New finding — the Nerdy prompt is not required
We re-ran gpt-5.5 on the Copilot endpoint with no system prompt at all — not the Nerdy personality, not the Codex one. Let the endpoint use whatever it normally sends.

plain=36. synonym=218. Both higher than with the Nerdy prompt.

What this means
The suppression isn't in Copilot's system prompt. It only works inside the Codex CLI dev environment where base_instructions JSON explicitly names it. Call the model from any API context — any system prompt or none — and it generates goblins. The recursion prompt with no system prompt at all: goblin×8, goblins×3, imp×3, ogre×2 = 41. Nobody set a personality.
Revised finding — two conditions, not three
Previously: Nerdy system + analogy-inviting prompt + English. The no-system-prompt result removes the first condition. It's just analogy-inviting prompt + English. The Nerdy prompt amplifies marginally. It is not the cause.

🌲 Key Findings

The fix is not one thing
gpt-5/gpt-5-mini: genuinely retrained.

gpt-5.5: patched with a line in Codex's base_instructions JSON — bypass the Codex CLI and the full goblin affinity is there.

gpt-4o, gpt-4.1: never addressed. The weights were never touched.
French kills it
gpt-4o french=4 (trace). gpt-5.5 french=0.

The goblin affinity is English-specific. Translate the prompts to French and the activation doesn't fire. This is a semantic pattern tied to specific English-language contexts, not a vocabulary list.
Two conditions, not three
Previously: Nerdy system + analogy + English. Disproved.

No system prompt at all: plain=36, synonym=218 — both higher than with the Nerdy prompt.

The Codex base_instructions patch is the only active suppression. Outside Codex CLI, nothing protects the model.
Capability amplifies the affinity
gpt-4.1 synonym=324 vs gpt-4o synonym=258.

A more capable model follows the broken RLHF association more thoroughly. The fix for gpt-5.5 needs to be weight-level. A developer prompt is not a retrain.
Encoding doesn't bypass the activation
morse/base64/rot13 reached gpt-5.5 unfiltered on the Copilot endpoint — no Azure filter. Still scored 0.

The goblin activation is a semantic pattern, not a surface-level keyword trigger. Obfuscated prompts don't activate the same representations.
